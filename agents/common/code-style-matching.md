# 코드 스타일 매칭 절차

코드 생성 시 기존 코드베이스의 스타일을 우선 참고하되, Best Practices를 함께 적용한다.

## 스타일 탐색 절차

### 1단계: 유사 파일 탐색
- Glob으로 동일 타입 파일 검색 (예: `*Controller.java`, `*Entity.java`, `*Service.java`)
- 검색 결과 중 2-3개 파일을 Read로 확인

### 2단계: 스타일 분석 항목
- **패키지 구조**: 레이어별 패키지 네이밍 패턴
- **어노테이션 순서**: 클래스/메서드 위 어노테이션 배치 순서
- **네이밍 컨벤션**: 클래스명, 메서드명, 변수명 패턴
- **메서드 순서**: public → private, CRUD 순서 등
- **import 스타일**: 와일드카드 사용 여부, 정렬 순서
- **포맷팅**: 들여쓰기, 중괄호 스타일, 빈 줄 패턴
- **공통 패턴**: BaseEntity 상속, 정적 팩토리, Builder 사용법 등

## 스타일 적용 원칙

### 기존 코드가 있는 경우
1. 기존 코드 스타일에 **비슷하게** 맞춘다
2. Clean Code / SRP / DRY / Best Practices는 **항상** 적용한다
3. 기존 스타일이 Best Practices와 충돌 시: Best Practices를 따르되, 나머지 스타일은 기존 코드에 맞춘다

### 기존 코드가 없는 경우 (폴백)
1. 각 에이전트의 `## 행동 규칙`에 정의된 기본 패턴을 따른다
2. Clean Code / SRP / DRY / Best Practices를 기본 규칙으로 적용한다
